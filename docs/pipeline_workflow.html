Phase 1 Raw Extraction

Architecture: 3 Independent Per-Label Agents

Each sample is processed by 3 parallel agents — one for Tissue, one for Condition, one for Treatment. Agents run as independent LLM calls via ThreadPoolExecutor. They do NOT share information with each other.

Prompt Architecture: System (KV-cached) + User (per-sample)

Each agent has a comprehensive system prompt that defines its role, expertise, rules, and examples. This is sent once and KV-cached by Ollama — subsequent calls only process the new metadata tokens (~30-40% latency reduction).

SYSTEM PROMPT (cached)
You are a biomedical tissue extraction specialist. Your sole task is to identify the biological tissue, cell type, or cell line from GEO sample metadata. YOUR EXPERTISE: - All human anatomical tissues and organs - Cell types: immune, epithelial, endothelial, stem... - Cell lines: HeLa, MCF-7, A549, HL-60, THP-1... - Brain subregions: hippocampus, prefrontal cortex... EXTRACTION RULES: 1. Most specific term (Alveolar Macrophages not Lung) 2. Priority: Cell Line > Cell Type > Tissue 3. Tumor samples → extract TISSUE not disease 4. Title Case. One answer only. EXAMPLES: "source: whole blood" → Whole Blood "MCF-7 cell line" → MCF-7 "prefrontal cortex tissue" → Prefrontal Cortex
USER PROMPT (per sample)
Title: Cutaneous psoriasis PsC 15 Source: whole blood Characteristics: tissue: whole blood; condition: Cutaneous psoriasis; gender: Female; batch: 1 Description: Biological replicate 15 of 20. Experiment: Psoriasis gene expression in blood Summary: Whole blood expression profiling...

Phase 1b NS Inference

Architecture: Per-GSE Inference with KV-Cached Context

For fields still "Not Specified" after Phase 1, per-GSE inference agents re-infer labels from the full experiment context. Each GSEInferencer owns one experiment and processes all its NS samples sequentially.

KV Cache Strategy

The GSE experiment metadata (title, summary, overall design) is placed in the system prompt. Ollama caches the KV tensors — so processing 50 NS samples from the same GSE only computes the system prompt ONCE. This gives ~40% latency reduction vs placing context in the user message.

Speed: ~0.5s/sample Strategy: GSE context as system prompt Impact: Condition resolution 91% → 97%

Phase 1c Full-Metadata Re-extraction

Architecture: No Truncation — Full Metadata for Hard Cases

Samples STILL "Not Specified" after Phase 1 + 1b are re-processed with NO character limits on Description (up to 2000 chars) and Summary (up to 1000 chars). This ensures no information loss for the hardest cases where disease/tissue info appears deep in the metadata text.

Why Phase 1c is Necessary

Phase 1 truncates metadata to keep latency low (~1.3s/sample): Description at 200 chars, Summary at 250 chars. But GEO summaries average 803 chars (73% lost) and descriptions average 765 chars (81% lost). Disease names and tissue descriptions can appear in the truncated portion.

Memory Architecture

Hierarchical Memory System

🧠 Working Memory (per-GSE)

Initialized at GSE start with experiment context. Accumulates resolution decisions as samples are processed. After resolving "breast tumor" → "Breast" for sample #1, samples #2-100 resolve instantly.

Scope: one experiment Speed: <1ms

💡 Episodic Cache (global)

In-memory dict mapping (column, raw_label) → canonical. Grows across GSEs. Later experiments resolve most labels without LLM calls because tissue/condition labels recur across studies.

Scope: all experiments Speed: <1ms

🔮 Semantic Retrieval (BioLORD-2023)

768-dimensional biomedical embeddings for all canonical clusters. BioLORD-2023 trained on UMLS knowledge graphs — understands "dermal" → "Skin" (0.86 sim) where general models fail (0.55).

Scope: vocabulary DB Speed: ~17ms

🗄️ Persistent Knowledge Store

biomedical_memory.db (SQLite): canonical clusters, cluster-to-variant mappings, knowledge graph triples (17,396 synonym relationships), episodic resolution logs across runs.

Scope: persistent Speed: ~1ms

Agent Definitions

Tissue Extraction Agent

gemma4:e2b ~1.3s/sample

Role: Identify biological tissue, cell type, or cell line from GEO metadata.

Expertise: Anatomical tissues, cell types (immune, epithelial, stem), cell lines (HeLa, MCF-7, A549, HL-60), brain subregions.

Key rule: For tumor samples, extract the TISSUE not the disease ("breast tumor" → Breast).

Priority: Cell Line > Cell Type > Tissue/Organ

Condition Extraction Agent

gemma4:e2b ~1.3s/sample

Role: Identify disease state, condition, or phenotype.

Expertise: Cancers, infections, genetic conditions, syndromes, phenotypes. Recognizes abbreviations (CAIS, AML, HCC). Understands that Normal/Control/Healthy are equivalent.

Key rule: Scan ALL metadata fields — disease info hides in Description, not just Characteristics.

Treatment Extraction Agent

gemma4:e2b ~1.3s/sample

Role: Identify drugs, compounds, interventions, or exposures.

Expertise: Pharmaceuticals, biological agents, gene knockdowns, physical treatments. Distinguishes treatments from lab protocols (Illumina, TRIzol are NOT treatments).

Key rule: Vehicle controls (DMSO, PBS) → "Untreated". Diseases are NOT treatments.

Collapse Agent (Phase 2)

gemma4:e2b BioLORD-2023 ~700ms/unique label

Role: Normalize raw labels to canonical cluster names using retrieval + reasoning.

Memory: Per-GSE working memory + global episodic cache. Knows experiment context.

Key rule: Preserve specificity — "Cerebellum" stays "Cerebellum", NOT collapsed to "Brain".
