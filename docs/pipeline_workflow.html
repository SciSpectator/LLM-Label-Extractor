2. Phase 1 Raw Extraction

3 Independent Per-Label Agents in Parallel

Each sample is processed by 3 parallel agents — Tissue, Condition, Treatment. Each agent has its own comprehensive system prompt (KV-cached by Ollama) defining its role, expertise, rules, and examples. Only the sample metadata is sent as the user message.

4. Phase 1c Full-Metadata Re-extraction

No Truncation — Complete Information for Hard Cases

Samples STILL "Not Specified" after Phase 1 + 1b are re-processed with NO character limits. Phase 1 truncates Description to 200 chars (avg 765, losing 73%) and Summary to 250 chars (avg 803, losing 73%). Phase 1c removes these limits.

Scientific justification: The pipeline first tries the fast approach (Phase 1), then context-aware inference (Phase 1b). Only for remaining hard cases (~3-5%) does it pay the latency cost of full metadata. The paper can state: "All samples with unresolved labels were re-processed with untruncated metadata to ensure no information loss."

6. Memory Architecture

🧠 Working Memory per-GSE

Initialized at GSE start with experiment context (title, summary). Accumulates resolutions as samples are processed. "breast tumor" → "Breast" resolved for sample #1 → samples #2-100 resolve instantly. Promoted to global episodic when GSE completes.

Scope: one experiment Speed: <1ms

💡 Episodic Cache global

In-memory dict: (column, raw_label) → canonical_name. Grows across GSEs. Later experiments resolve most labels without LLM calls because tissue/condition labels recur across studies. Also caches normalized variants (lowercase, no-hyphen, title-case).

Scope: all experiments Speed: <1ms

🔮 Semantic Retrieval BioLORD-2023

768-dim biomedical embeddings for all canonical clusters. Trained on UMLS knowledge graphs — understands biomedical relationships that general models miss. "dermal" → "Skin" at 0.86 (vs 0.55 with MiniLM).

Scope: vocabulary DB Speed: ~17ms

🗄️ Persistent Store SQLite

biomedical_memory.db: 1,459 Tissue + 2,704 Condition + 3,105 Treatment canonical clusters. 17,396 knowledge graph triples (synonym relationships). Episodic resolution logs across runs. Curated from biomedical ontology sources.

Scope: persistent Speed: ~1ms

7. Agent Definitions

🤖 Tissue Extraction Agent

gemma4:e2b ~1.3s/sample

Role: Identify biological tissue, cell type, or cell line.

Expertise: Anatomical tissues, cell types (immune, epithelial, stem), cell lines (HeLa, MCF-7, A549, HL-60), brain subregions.

Key rules: Priority: Cell Line > Cell Type > Tissue. For tumor samples extract TISSUE not disease ("breast tumor" → Breast). Scan ALL metadata fields.

🤖 Condition Extraction Agent

gemma4:e2b ~1.3s/sample

Role: Identify disease state, condition, or phenotype.

Expertise: Cancers, infections, genetic conditions, syndromes, phenotypes. Recognizes abbreviations (CAIS, AML, HCC). Normal/Control/Healthy → "Control".

Key rules: Scan ALL fields — disease info hides in Description, not just Characteristics. Mutations imply disease (CFTR D508 = cystic fibrosis).

🤖 Treatment Extraction Agent

gemma4:e2b ~1.3s/sample

Role: Identify drugs, compounds, interventions, exposures.

Expertise: Pharmaceuticals, biological agents, gene knockdowns, physical treatments. Distinguishes treatments from lab protocols and diseases.

Key rules: Vehicle controls (DMSO, PBS) → "Untreated". Diseases are NOT treatments. Lab protocols (Illumina, TRIzol) are NOT treatments.

🤖 Collapse Agent (Phase 2)

gemma4:e2b BioLORD-2023 ~700ms/unique memory-enabled

Role: Normalize raw labels to canonical cluster names using retrieval + reasoning.

Memory: Per-GSE working memory + global episodic cache. Sees experiment context + recent resolutions.

Key rules: PRESERVE specificity (Cerebellum ≠ Brain). Only normalize naming. Use experiment context to disambiguate.
